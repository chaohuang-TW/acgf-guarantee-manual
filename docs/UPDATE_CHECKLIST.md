# 農業信用保證業務作業手冊新版更新檢核清單

## A. 收到新版前

- [ ] 確認新版PDF可全文公開。
- [ ] 確認版本年月。
- [ ] 確認農漁會版或其他版本別。
- [ ] 保留原始檔，不重新壓縮。
- [ ] 計算SHA-256。
- [ ] 確認PDF實體頁數。
- [ ] 確認是否已有相同版本ID。
- [ ] 確認Git tag尚未存在。
- [ ] 確認GitHub Release尚未存在。

## B. 建立版本資料

- [ ] 建立新的版本ID，例如115-10。
- [ ] 新增versions.json紀錄。
- [ ] 舊版本isCurrent改為false。
- [ ] 舊版本status改為archived。
- [ ] 新版本isCurrent設為true。
- [ ] 不刪除舊版site路徑。
- [ ] 不覆蓋舊版PDF。
- [ ] 不覆蓋舊版搜尋索引。
- [ ] 記錄新PDF頁數、SHA及無文字層頁面。

## C. 擷取與目錄

- [ ] 優先使用PDF文字層。
- [ ] 不使用OCR。
- [ ] 重新核對正式目錄。
- [ ] 不沿用舊版頁碼映射。
- [ ] 核對四大篇是否變更。
- [ ] 核對附錄數量。
- [ ] 核對書表及格式編號。
- [ ] 核對新增、刪除、修正項目。
- [ ] 不自行修正原文。

## D. 建置

- [ ] 執行extract_manual.py。
- [ ] 執行build_site.py。
- [ ] 產生版本專屬頁面。
- [ ] 產生版本專屬搜尋索引。
- [ ] 更新根目錄目前版本。
- [ ] 更新版本紀錄頁。
- [ ] 更新CHANGELOG。
- [ ] 建立該版本Release Notes。
- [ ] 更新SHA256SUMS.txt。

## E. 驗證

- [ ] 執行audit_content.py。
- [ ] 執行validate_site.py。
- [ ] 比對原始PDF。
- [ ] 抽查至少20個位置。
- [ ] 核對金額、成數、日期、年限、格式編號。
- [ ] 測試20項關鍵字。
- [ ] 測試PDF頁碼定位。
- [ ] 測試深層連結。
- [ ] 測試390、768、1024、1440px。
- [ ] 測試列印。
- [ ] Console error為0。
- [ ] Network 404為0。
- [ ] 無外部API。
- [ ] 無AI。
- [ ] 無OCR。
- [ ] 無GA4。
- [ ] 無敏感資料。

## F. 發布

- [ ] Commit並push到main。
- [ ] 等待GitHub Pages成功。
- [ ] 實際檢查正式網站。
- [ ] 建立annotated tag。
- [ ] Push tag。
- [ ] 建立GitHub Release。
- [ ] 附上原始PDF。
- [ ] 附上SHA256SUMS.txt。
- [ ] 確認Release網址。
- [ ] 確認舊版仍可開啟。
- [ ] 確認舊版顯示已非最新版。
- [ ] 記錄最終commit hash。

## G. 發布後

- [ ] 保存發布紀錄。
- [ ] 記錄已知限制。
- [ ] 不刪除舊版。
- [ ] 若發現內容錯誤，以新commit修正，不改寫歷史tag。
- [ ] 若原始PDF正式撤回，先保留紀錄並清楚標示，不擅自消失。
